### 🔍 Overview
This component of the project focuses on comparing the original customer data (`telco_prep.csv`) with the results generated by the Sentiment Analysis engine (`sentiment_results.csv`). The goal was to ensure that customer identifiers remained consistent and that the Churn status aligned perfectly across both files.

### 📊 Dataset : <a href='https://www.kaggle.com/datasets/beatafaron/telco-customer-churn-realistic-customer-feedback'> telco_prep </a>

### 📊 Performance & Accuracy
By comparing the ground truth from the preprocessed data against the analyzed output, we generated a **Confusion Matrix** to visualize the alignment.

#### **Key Metrics:**
* **Data Alignment Accuracy:** `100%`
* **Consistency Check:** All `7,032` overlapping customer records showed matching churn status.
* **Error Rate:** `0.00%` (No False Positives or False Negatives during the merge).



### 📈 Visualizing the Result
The following visualizations were used to confirm the results in Google Colab:

1.  **Churn Distribution:** A side-by-side comparison of the number of customers who stayed vs. those who left.
2.  **Accuracy Heatmap:** A Confusion Matrix proving that every "Yes" and "No" in the source file correctly mapped to the results file.
